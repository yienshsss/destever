### 1. 로컬 테스트 환경 안정화 (Windows)

**문제:** 라이브 DB를 raw 파일로 복사하면 타임아웃 발생, URL 직렬화 미치환으로 로컬 테스트 버그 다수 발생

**해결:**
- `tools/local-test-site.ps1`에 URL 직렬화-safe 치환 추가 (up/repair-urls 후 자동 실행)
- DB 동기화 방식을 raw robocopy → SSH mariadb-dump import로 교체 (`refresh-db` 액션)
- `up`은 기존 로컬 스냅샷 재사용, DB 없을 때만 자동 fresh import
- `repair-urls` 액션 추가 (이미 떠 있는 환경에서 URL만 재보정)

**검증:** `localhost:8160` 200 OK, Done List 비로그인 접근 로그인 페이지 리다이렉트 확인

### 2. Avada 부모 테마 drift 정리

**문제:** 4/28 긴급 대응 중 Avada 부모 테마를 라이브에서 직접 수정 → Git 추적 범위 밖

**조사 결과:**
- `Avada/category.php`: 이미 `Avada-Child/category.php`가 덮어쓰고 있어 무영향
- `Avada/project-b-custom.php`: 어디서도 로드되지 않는 죽은 파일
- `Avada/header.php`: 커스텀 오버레이 메뉴 전체 포함, child override 없어 활성 drift

**해결:**
- `Avada/header.php` → `Avada-Child/header.php`로 복사 (Git 추적 범위 편입)
- `Avada/project-b-custom.php` 라이브에서 삭제
- `tools/sync-theme-to-live.ps1`에 `mu-plugins` 동기화 추가
