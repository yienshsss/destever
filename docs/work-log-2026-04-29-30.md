**검증:** `localhost:8160` 200 OK, Done List 비로그인 접근 로그인 페이지 리다이렉트 확인

### 6. PROS 오버레이 메뉴 복원 + BLOG 보드 목록 썸네일 적용

**문제 1:** 오버레이 메뉴에서 `PROS` 하위 `커미션 / 리퀘`가 보이지 않았고, 로컬은 child `header.php`가 아닌 부모 테마 헤더를 계속 참조하고 있었음

**해결:**
- `wp-content/themes/Avada-Child/header.php`에 권한 사용자 전용 `커미션 / 리퀘` 항목 복원
- 로컬 테스트 런타임 `.local/wp-content`에 child `header.php`가 실제 반영되도록 오버레이 복사

**문제 2:** `BLOG` 하위 카테고리 목록에 썸네일 열을 넣으려 했지만, `blog-notes` 등은 `Avada-Child/category.php`가 아니라 Avada 기본 archive/post-cards 그리드를 타고 있었음

**해결:**
- `wp-content/themes/Avada-Child/functions.php`의 보드 템플릿 강제 조건에 `blog` 루트 카테고리 추가
- `wp-content/themes/Avada-Child/category.php`에서 `BLOG` 하위이면서 `Done List`가 아닌 보드 목록에만 썸네일 열/모바일 축소 스타일 적용
- `PROS`, 갤러리 뷰, `Done List` 뷰는 기존 분기 유지

**검증:** `localhost:8160/category/blog-notes/` 응답 HTML에서 `pb-board__row--with-thumb`, `pb-board__thumb`, `pb-archive__title` 확인, 브라우저에서 썸네일 표시 확인

### 7. BLOG 보드 목록 인라인 관리 1~2단계

**문제:** 일반 게시판 목록에서 글을 바로 수정/삭제할 수 없었고, 기존 목록 행 구조는 액션 버튼을 붙이기 어려운 전체 링크 형태였음

**해결:**
- `wp-content/themes/Avada-Child/category.php`의 일반 게시판 행을 전체 `<a>` 래핑에서 `div + 제목 링크` 구조로 변경
- 오른쪽 끝 카테고리 텍스트 열 제거 후 수정/삭제 아이콘 전용 액션 칸으로 교체
- 로그인 + 해당 글 편집 권한이 있는 사용자에게만 수정/삭제 버튼 노출
- 삭제 버튼은 WP REST API로 즉시 삭제 후 목록 DOM에서 제거되도록 구현
- 수정 버튼은 Quill.js 기반 모달 에디터를 열어 제목/본문을 불러오고, 텍스트 편집 후 저장하면 REST API로 반영되도록 구현
- 이번 단계에서는 이미지 업로드/정렬/대표 이미지 지정은 제외하고 텍스트 편집 흐름까지만 완료

**검증:** 로컬 `blog-notes` 목록에서 아이콘 버튼 노출 확인, 수정 모달 오픈 및 제목/본문 저장 반영 확인
